# Double-Click to Preview Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Change the Lightbox open trigger from single-click to double-click on asset cards. Single-click selects (DetailPanel / BottomSheet opens); double-click opens the preview Lightbox (image/video only). Mirrors Finder / Explorer / Drive / Bridge conventions.

**Architecture:** Pure frontend interaction change. `App.tsx`'s `handleSelectAsset` is split into two useCallbacks (`handleSelect` = single-click, `handleOpen` = double-click). Three card/row components accept a new optional `onDoubleClick` prop and wire it to the native `onDoubleClick` event. Keyboard activation splits: `Space` = select, `Enter` = open (or fall back to select for non-previewable types). No backend change. No new reducer action. No new dependency.

**Tech Stack:** React 19, TypeScript 5.6, Vitest 2, Testing Library (`@testing-library/user-event` v14 for `dblclick`), Playwright (visual verification only).

**Spec:** `docs/superpowers/specs/2026-06-07-double-click-lightbox-design.md`

```tsx
function onKeyDown(e: KeyboardEvent<HTMLDivElement>) {
  if (e.target !== e.currentTarget) return;
  if (e.key === 'Enter') {
    e.preventDefault();
    // Enter mirrors mouse double-click: open the preview if a handler
    // is provided, otherwise fall back to the single-click action.
    if (onDoubleClick) onDoubleClick();
    else onClick();
  } else if (e.key === ' ') {
    e.preventDefault();
    onClick();
  }
}
```

## Generalization rules (apply during implementation)

1. **A click action and a confirm action should be different events.** Single-click = low-cost, low-commitment gesture (select, reveal in side panel). Double-click = higher-commitment gesture (fullscreen overlay). Conflating them is the bug we're fixing.
2. **Type-specific branches belong at the consumer, not in the click handler.** The card/row is type-agnostic. App.tsx's `handleOpen` decides whether to open the Lightbox based on the asset's type. The card just forwards the event.
3. **Re-dispatching the same action is a reducer no-op, not a bug.** The browser fires two clicks per dblclick. We let both through. The reducer's `SELECT_ASSET` for the same id produces the same state, so React skips the re-render.
4. **Keyboard activation should mirror mouse activation.** Double-click → `Enter`. Click → `Space`. This rule covers future components without re-deciding.
